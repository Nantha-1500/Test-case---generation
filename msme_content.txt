141. Accessing to LENDperfect application
142. This section explains bank users accessing of LENDperfect application.
143. Business case for Accessing the application
144. Bank users will access the LENDperfect application strictly via the internal bank network through a URL provided by the DIT team. The system supports secure user login using existing credentials, validated against the bank’s authentication database.
145. LENDperfect also offers robust credential management, including features for:
146. User Login with IP logging and session tracking (Refer to User Loan)
147. Password Change, validated via OTP sent to registered mobile and email (Refer to Change Password)
148. Forget Password with OTP-based verification and password policy enforcement (minimum 8 characters, alfa numeric, and at least one uppercase character) ( Refer to Forget Password)
149. These features are fully aligned with the bank’s IT security. They ensure secure access while empowering users to manage their credentials independently.
150. User Login- LENDperfect- application
151. As per bank guidelines, applications can be accessed only through the bank’s internal website. The Bank DIT team will provide the URL, which needs to be configured by the development team.
152. Users must manually enter the approved URL provided by the DIT team.
153. The login screen allows authorized branch users to access the LENDperfect system. Users can log in using their existing user ID and password. Upon entering their credentials, the system will validate the details and grant access if the entered information is correct. Otherwise, the system will display an error.
154. Positive Scenario: Successful Login
155. User Action:
156. The user enters the approved URL in the browser and accesses the login screen.
157. The user provides a valid User ID and Password on the login screen.
158. Backend System Computation:
159. The system validates the entered credentials against the bank’s authentication database.
160. If the credentials are correct, access is granted.
161. User Access Granted:
162. The user is redirected to the Home Screen of the LENDperfect application.
163. Security Confirmation:
164. The system logs the successful login, capturing details such as:
165. IP address
166. Timestamp
167. User session details
168. Negative Scenario: Login Failure
169. User Action:
170. The user enters the approved URL and attempts to log in.
171. Backend System Computation Error:
172. The system fails to validate the User ID or Password due to incorrect credentials.
173. The user attempts login with an expired or deactivated account.
174. Access Denied:
175. The system displays an error message:
176. “Invalid credentials. Please try again, with valid credentials.”
177. Logging and Monitoring:
178. The system logs the failed login attempt, capturing:
179. IP address
180. Timestamp
181. Error details
182. Account Lock:
183. If the login attempt is wrong for 3 times for invalid password, then account gets locked, user needs to contact administrator manually for unlocking.
184. The system locks the users in accessing the application if the tries for set configured times.

348. Branch Officer-Initiated MSME Loan Application: Workflow Handling
350. When a Branch Officer initiates an MSME loan application in LendPerfect LOS, the system evaluates the processing path at the time of application creation based on delegation powers.
351. Workflow Logic:
352. The system uses configured global variables to determine the appropriate processing level.
353. The routing decision is based on:
354. Requested Loan Amount
355. Proposed Loan Amount
356. These values are mapped against the delegation of powers matrix to decide whether the application should be:
357. Processed at the Branch level, or
358. Routed to MSME Loan Point (referred to as MLP)
359. System Behavior:
360. The routing logic is triggered immediately upon application creation, and the processing path is locked based on loan amount values.
361. However, during the next workflow stage, the identified path can be forwarded to the next for further review, and the user may send the proposal to MLP even if it is currently at the Branch Manager review stage(During that stage, the user will have access to the full application and can complete the entire application before forwarding it to the next workflow, as configured in the system by the bank.)
362. The system will dynamically enable or disable specific workflow roles based on the identified processing level during the application creation.
